# African Marketplace API Documentation

## **SCHEMA**

### ‚≠ê users

| Field            | Type    | Metadata                            |
| ---------------- | ------- | ---------------------------------- |
| id               | integer | _primary key_ and _autoincrements_ |
| email            | string  | _required_ and _unique_            |
| password         | string  | _required_                         |
| first_name       | string  | _required_                         |
| last_name        | string  | _required_                         |
| shop_name        | string  | _required_                         |
| location_id      | integer | _required_                         |
| role_id          | integer | _required_ and _autogenerated_     |

‚ùó ```location_id``` and ```role_id``` are foriegn keys!

---
### ‚≠êroles

| Field      | Type    | Metadata                           |
| ---------- | ------- | ---------------------------------- |
| id         | integer | _primary key_ and _autoincrements_ |
| role       | string  | _required_ and _unique_            |

‚ùó ```role``` can be ```owner``` (id: 1) or ```shopper``` (id: 2).

---
### ‚≠êlocations

| Field      | Type    | Metadata                           |
| ---------- | ------- | ---------------------------------- |
| id         | integer | _primary_ and _autoincrements_ |
| location   | string  | _required_ and _unique_            |

‚ùó ```location``` can be ```KEN``` (id: 1), ```TZA``` (id: 2), ```RWA``` (id: 3), or ```UGA``` (id: 4).

### ‚≠êproducts

| Field            | Type    | Metadata                           |
| ---------------- | ------- | ---------------------------------- |
| id               | integer | _primary_ and _autoincrements_ |
| item_name        | string  | _required_                         |
| price            | float   | _required_                         |
| description      | string  | _optional_                         |
| user_id          | integer | _required_                         |
| category_id      | integer | _required_                         |

‚ùó ```user_id``` and ```category_id``` are foreign keys!

### categories

| Field          | Type    | Metadata                           |
| ----------     | ------- | ---------------------------------- |
| id             | integer | _primary_ and _autoincrements_ |
| category_name  | string  | _required_ and _unique_            |

‚ùó These are ```animal products``` (id: 1), ```cereals``` (id: 2), ```clothing and shoes``` (id: 3), ```cosmetics``` (id: 4), ```fruits``` (id:5), ```seeds and nuts``` (id:6), ```vegetables``` (id:7), and ```other``` (id:8).


## **ENDPOINTS**
üåçroot url :  https://tt-202-african-marketplace.herokuapp.com

### ‚≠êPOST /api/auth/register/owner
*Adds user with owner role to the database*

request
```json
{
	"email": "jamboojam@email.com",
	"password": "iL0<3j@M",
	"first_name": "Jami",
	"last_name": "Boo",
	"shop_name": "Jam Boo Jams",
	"location_id": "4"
}
```

response:
```json
{
    "message": "Thank you for registering, Jami!", 
	"email": "jamboojam@email.com",
	"shop_name": "Jam Boo Jams",
}
```

### ‚≠êPOST /api/auth/login
*Logs in a registered user with valid credentials and gives a token, if sucessful.*

request:

```json
{
	"email": "jamboojam@email.com",
	"password": "iL0<3j@M",
}
```

response:
```json
{
    "message": "Welcome, Jami",
    "id": "1",
	"token": "wow.thats.a.long.token",
}
```

### ‚≠êPOST /api/auth/add-product
‚ùó *Due to project restraints, user does not have to be logged in for this to work.*

*Will add a product to the database. ```decription``` can be null.*

request:
```json

    "category_id": 5, 
    "price": 10.50,
    "description": "odd but sweet" 
}
```

```json
        "id": 13,
        "item_name": "odd fruit", 
        "user_id": 1,
        "category_id": 5, 
        "price": 10.50,
        "description": "odd but sweet" 
    }
}
```
